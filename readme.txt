Описание проекта:
Есть выбранная группа(-150622463), из которой при запуске программы забирается 99 постов.
Они сохраняются в data->raw.json
Дальше программа формирует словарь №1, где ключом является id поста, а значением его текст. После этого, с помощью
регулярных выражений определяется является ли пост рекламным или нет. (В данном случае такое решение дает хорошие
результаты, что связано с спецификой выбранной группы:
    - С одной стороны, регулярные выражения, использованные для выявления рекламы, редко встречаются вне связи с рекламными постами.
    - С другой стороны, рекламные посты содержат много текста, в которых много типичных для рекламных постов слов).
Результат анализа вносится в словарь №2, где ключом является id поста, а значением его текст "это реклама"/"это не реклама".
Полученный словарь передаётся в очередь с названием "post_add_result"
(rabbitMQ http://localhost:15672/  логин:guest  пароль:guest)
Из очереди сообщение не забирается.
